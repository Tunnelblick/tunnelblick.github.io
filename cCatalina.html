Tunnelblick and macOS Catalina

The following issues have been identified when using Tunnelblick 3.7.8beta02 on macOS Catalina beta version 2:

(Last updated 2019-06-24 17:00 UTC.)

"System Requirements Not Met: the tmp system folder (/tmp) is not secure" appears each time Tunnelblick is launched.

See Permissions check is broken for MacOS Catalina (/tmp folder stored in System Volume).

Workaround: Click "Continue".

This problem will be fixed in the next beta release of Tunnelblick.

"Installation or repair took too long or failed" appears when trying to install Tunnelblick.

This message may also occur when installing any version of Tunnelblick in a virtual machine (e.g. using VirtualBox, Parallels, or VMWare).

Workaround: Copy the Tunnelblick disk image to the boot drive of the virtual machine and install from the copy.

Catalina refuses to open a downloaded Tunnelblick disk image, saying it is "from an unidentified developer".

This is either a problem with the way Tunnelblick 3.8.0beta02 is signed and notarized, or a bug in Catalina. It doesn't happen on macOS Mojave and it doesn't happen on Catalina using older versions of Tunnelblick which are not notarized.

Workaround: Open the disk image by Control-clicking it and selecting "Open". You'll again be told it is from an unidentified developer, but you can click the "Open" button to open the disk image. Once the disk image has been opened, you can double-click the Tunnelblick icon to install Tunnelblick.
